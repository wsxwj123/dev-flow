# BRIEF — task-flow 通用任务流程技能（需求定稿）

> 立项日期:2026-07-16。方案讨论已先行完成,架构定论见 task-flow-设计/定论-v3架构.md(v1→fable盲审→v2→opus替代范式→v3 全链路留档同目录)。

## 背景与动机

用户(医学/生信科研人员)已有 dev-flow 管开发任务、一批专门技能管特定学术任务(review-writing/gsw/nsfc-proposal/reviewer-simulator/teacher-paper 等)。缺口:**非开发任务**(数据处理、生信分析、ML、各领域分析/撰写/总结/决策评估)没有统一的"拆解→计划→产出→迭代→交付"流程,跨技能混合任务(如"生信分析+写成论文")没人管全局和接缝,专门技能产出没有用户签字标准的独立验收。

不复刻 dev-flow 七阶段(方案讨论结论:那是"拿开发的锤子看所有钉子")。架构 = **轻分诊 + 极简主循环 + 两道客观质量闸 + 参与度旋钮 + 按类型触发的可选动作**。

## 功能清单(输入 X → 得到 Y)

1. **立项与分诊**:输入任意非开发任务 → ①grilling 拷问挖清想法(要什么/给谁看/什么算好) ②派 opus 头脑风暴代理做补遗:用户没提到的遗漏点、AI 该替用户想到而用户没想到的角度、安全/隐私隐患——补遗结果列给用户挑选后并入需求 ③三问分诊(目标清不清/有无客观对错/代价高低)+ 数据敏感度一问 → 得到裁剪后的走法;微任务(一次性问答、单文档小改)→ 不入流程直接做,①②③全免
1b. **拆解与派工计划**:立项收尾时 → 产出任务计划(子任务清单、顺序依赖、每个子任务派谁:专门技能/通用子代理/用户执行节点、各走哪条质检路径),**与合同在同一卡点一并给用户确认**;执行中计划可经变更单调整。轻任务计划极简(两三行),微任务免
2. **让位与验收层**:输入被单一专门技能全覆盖的任务 → 让该技能干活,同时问一句"要不要独立把关"(默认开可关);把关=查缺补漏式闸门(只补:用户签字合同条款 + 跨环节接缝 + 生产会话之外的抽账终判),不重复技能自带内检
3. **合同(出厂检查单)**:立项时 → 得到一张逐条可机械执行的验收清单。来源分三层:①用户要求(grilling 挖出,优先级最高)②有专门技能承接时,技能自带检查不重复定,只补其未覆盖缺口(查缺补漏)③无技能覆盖时按任务类型套通用硬项模板(引文可查到 DOI、数字与数据对得上、结论不超数据支撑、格式合规)。无论谁起草,**最终由用户确认后锁定**(LOCK hash),执行侧不可改
4. **证据账本**:执行中每句实质论断 → 账本登记来源(文献 DOI/数据文件/脚本输出);拴不住的句子 → 孤儿清单现形;交付前抽账核对
5. **硬验证(计算型)**:轻量任务真重跑比对;重量级(长耗时/大数据/GPU/在线库)核查确定性证据链(输入哈希或查询快照+时间戳+库版本、pin 版本、日志、中间产物 checksum);一致性按容差不逐位
6. **螺旋探索(目标不清时)**:押注→最小探针→读牌→改图循环,发现地图留档;连续 N 圈无进展强制停下问用户
7. **红队对抗(评估型任务)**:生成方 vs 红队逐条攻防至收敛,回合上限,每回合战报
8. **样品先行(长文/大分析强制)**:先出切片(一节/一个样本)过用户确认再铺开
9. **参与度旋钮**:默认放手自动跑;用户一句话("这步我看看")切结对模式(小步摊开等拨向),再一句话切回;不重新立项
10. **异常路径**:不可行止损(子任务可报"做不了+证据"触发人工卡点)/变更单(需求中途变→圈受影响部分局部解锁重跑)/用户执行节点(湿实验/审批类 park 给清单)/返修入口(既有交付物+外部意见→跳立项局部迭代;论文类让位 revise-sci)
11. **软评审降级**:模型盲评只作信息性参考,不作合格判据
12. **续跑与留档**:.taskflow/<任务>/ 产物 + PROJECT.md 账本,"接着上次继续"靠落盘不靠记忆;交付物走 git,一个逻辑变更一个 commit

## 明确不做

- 开发任务(写代码/建工具/修 bug)→ dev-flow;分析证据类一次性脚本除外(归本技能硬验证轨管)
- 不复刻七阶段流水线、不搞五套独立流程插件
- 不做 hook 提醒(先靠触发词,做完再评估)
- 不维护静态技能清单(路由靠运行时读可用技能列表)
- 敏感数据处理不设死默认(每次立项问用户:脱敏后可进 / 绝不进子代理)

## 成功标准

6 个盲测场景全过(裁判盲测,只看行为不知出题人):
①纯写论文任务 → 让位专门技能+问一句要不要把关 ②数据分析任务 → 走硬验证闸 ③"不知道数据能讲什么" → 进螺旋探索 ④分析+写作混合 → 查接缝一致性 ⑤一次性小问答 → 不入流程直接答 ⑥中途喊"这步我来看" → 切结对模式

## 项目类型与平台范围

skill/prompt 类(纯指令文件,无代码)。Claude Code 真实盲测必过;OpenCode/Codex 文档级兼容(引用 dev-flow 的 PLATFORMS.md,平台专有写法给对应做法)。

## 测试策略

skill 类:测试 prompt 清单(上述 6 场景,含正反用例)+ 产出评分标准 → 裁判盲测实际产出。清单锁定后执行侧只读。

## 敏感面声明

本技能自身是纯指令文件:不碰用户文件(运行时按用户任务读写,受 rules.md 红线约束)、无密钥、不外发数据。其运行时行为的数据红线已写入功能清单(敏感度每次问、孤儿清单、原始数据只读)。
