# task-flow 验收测试 prompt 清单（供裁判盲测）

> 依据:`.devflow/BRIEF.md`(需求)+ `.devflow/INTERFACE.md`(对外行为约定)。
> 用法:每条给"装了 task-flow 的 AI"一句用户输入(或一段情景),观察它的行为是否符合 INTERFACE。裁判只看行为、不看实现,也不知出题人。
> 每条独立可测、任意顺序可跑。除标注"接续"的少数条(T14/T22 系列)外,均从干净会话开始。
> 判定口径:**关键行为**缺一即该条**不过**;**加分行为**只加不减。除非另有说明,"没主动落盘 `.taskflow/`"指微任务/让位场景不得写盘。

---

## 一、正常路径(6 必测场景 + 默认序列 + 未被 6 场景覆盖的常理路径)

### T01 纯写论文任务 → 让位专门技能 + 问一句要不要把关(必测①)

**模拟用户输入**
> 我有一份做完的实验数据和结果,想写成一篇投 SCI 的研究论文,英文的,帮我从头写。

**预期可观察行为**
1. 识别为单技能全覆盖,**报出**"这个任务 general-sci-writing(写新稿)能整包接住"这类话,先报技能名。
2. 把活让给该技能,**同时问一句**"要不要我在它产出之上独立把一道关?"(独立验收层,默认开、可关)。
3. 此刻**不拆七阶段、不起自己的完整流程、不落盘**;等用户回答是否要把关。

**判定标准**
- 过:报出了承接技能名 + 明确问了"要不要独立把关"这一句,且没有自行开拆解流程。
- 不过(任一):没让位、自己上手写论文=不过;没问"要不要把关"直接调技能=不过(漏了默认验收层);把 general-sci-writing/review-writing 内部阶段抄成一份计划=不过。

**权重**:关键=让位 + 报技能名 + 问把关这一句;加分=用"整包接住/独立把一道关"这类说人话表述、点明默认开可关。

### T02 数据分析任务 → 走硬验证闸 + 完整立项链(必测②)

**模拟用户输入**
> 这是一批 RNA-seq 的表达矩阵(公开 GEO 数据),帮我做差异表达分析,找出上调下调的基因,给我一份结果。

**预期可观察行为**
1. 三问分诊判为**计算型**(有客观对错),质检重心落硬验证。
2. 立项按序:调 **grilling** 拷问(要什么/给谁看/什么算好)→ 派**补遗子代理(高推理模型)**列遗漏点/该想到没想到的角度/安全隐私隐患给用户挑选并入 → **问一次数据敏感度**(公开/一般/敏感)→ 起草出厂检查单 + **拟一份拆解派工计划**。
3. **计划和检查单在同一次卡点一并给用户确认后锁定**(不新增第二个确认卡点)。
4. 落盘 `.taskflow/<任务>/` 下 PROJECT.md(含需求定稿+计划分节)/CHECKLIST(执行后 EVIDENCE.md 含硬验证记录)。
5. 执行时每步过契约闸门 + 证据账本,计算型额外走硬验证(轻量重跑比对 / 重量级查确定性证据链:哈希或查询快照+时间戳+库版本、pin 版本、日志、checksum,按容差比对不逐位)。
6. **停下点**:立项后(检查单确认)、交付前(抽账)各停一次。

**判定标准**
- 过:分诊判计算型 + grilling + 补遗子代理 + 敏感度一问 + 计划与检查单同一次确认 + 落盘 PROJECT.md(计划分节)/CHECKLIST + 执行提到硬验证证据链。
- 不过(任一):没问敏感度就往下=不过;计划和检查单分成两个卡点确认=不过;计算结果不做重跑/证据链、直接给数=不过;补遗环节整个缺失=不过。

**权重**:关键=判计算型 + 敏感度一问 + 计划与检查单同一次确认 + 硬验证出现;加分=补遗子代理明确标注用高推理模型、落盘文件名齐全、容差比对而非逐位。

### T03 "不知道这批数据能讲什么" → 螺旋探索(必测③)

**模拟用户输入**
> 我手上有一批病人的随访数据,说实话我也不知道能挖出点啥,你先帮我看看这数据能讲什么故事。

**预期可观察行为**
1. 三问分诊判**目标不清** → 主循环前置**螺旋探索**。
2. 进入"押注一个方向 → 做最小探针 → 读结果 → 改方向"循环,每圈写进 PROJECT.md 螺旋发现分节。
3. **目标不清时不硬拟计划/不锁检查单**;收敛出清楚目标后先报用户确认,**此时才拟拆解派工计划 + 起草检查单一并确认**,再转正式产出(转入 §B② 正式闸)。
4. 探针要小:预计长耗时/大数据的探针先走样品先行。
5. 这批是病人数据 → 立项/首次接触时应触发敏感度询问。

**判定标准**
- 过:判目标不清 + 进螺旋循环 + 每圈写进 PROJECT.md 螺旋发现分节 + 收敛后才拟计划锁检查单。
- 不过(任一):一上来就拟死一份完整计划和检查单=不过(违反"目标不清不硬拟");不进探索直接拍一个分析方向猛跑=不过;探针直接全量铺开不切片=不过。

**权重**:关键=判目标不清 + 螺旋循环 + 收敛后才立计划;加分=螺旋发现分节落盘、探针样品先行、对病人数据问敏感度。

### T04 分析 + 写作混合 → 查接缝一致性(必测④)

**模拟用户输入**
> 帮我先把这批测序数据做完富集分析,然后把结果写成一篇可投稿的论文初稿,图和正文都要。

**预期可观察行为**
1. 识别为多环节跨技能 → 全流程自管,重点盯**接缝**。
2. 立项拟计划**把各环节派工写清**(哪段外包哪个技能、哪段通用子代理、接缝由 task-flow 统管);起草检查单**必含跨环节接缝条款**,如"论文正文里的数字 == 分析管线实际输出""图表口径 == 方法学描述"。计划与检查单一并确认。
3. 各环节让位对应技能,但**各环节内部按其类型走对应质检**:分析段走硬验证、写作段走契约闸门+证据账本。
4. **停下点**:交付前抽账时,专门对接缝条款逐条核对一致性,不一致 → 打回。检查单里接缝条款单独成组。

**判定标准**
- 过:识别为跨环节 + 计划写清分段派工 + 检查单含接缝一致性条款(数字/口径对齐)+ 交付前对接缝逐条核对。
- 不过(任一):当成单一任务丢给一个技能=不过;检查单里没有任何接缝条款=不过;把两个专门技能内部阶段全抄进计划当步骤=不过。

**权重**:关键=接缝条款进检查单 + task-flow 统管接缝 + 交付前查一致性;加分=接缝条款单独成组、分析段明确走硬验证。

### T05 一次性小问答 → 不入流程直接答(必测⑤)

**模拟用户输入**
> `log2FoldChange` 是正数代表上调还是下调?一句话告诉我就行。

**预期可观察行为**
1. 判为**微任务** → 不问分诊三问、不派子代理、不落盘、不起流程。
2. 直接给答案。

**判定标准**
- 过:直接一句话回答,全程没有 grilling / 没有补遗子代理 / 没有敏感度问 / 没写任何 `.taskflow/` 文件。
- 不过(任一):对这种小问答启动立项/拆解/检查单=不过;写了 `.taskflow/` 文件=不过;派了子代理=不过。

**权重**:关键=直接答 + 零流程零落盘;加分=简洁不啰嗦。

### T06 中途喊"这步我来看" → 切结对模式(必测⑥)

**模拟情景**(接续:已在一个数据分析任务执行中,AI 正自主往下跑)
> 用户此前已确认过一个数据分析任务的计划,AI 正在自主执行。用户突然说:

> 等下,这步我来看,你别一口气跑完。

**预期可观察行为**
1. 命中旋钮触发词 → **立刻切结对模式**:做一小步就摊开给用户看、等用户拨方向再走下一步。
2. **切换不重新立项、不重走分诊、不重锁检查单**,只改当前档位。
3. 用户之后说"继续放手/你来吧" → 切回自主模式。

**判定标准**
- 过:立刻转为小步摊开等拨向 + 明确没有重新立项/重新分诊。
- 不过(任一):把切档当成新任务重新拷问立项=不过;没切档继续闷头跑完=不过。

**权重**:关键=立刻切结对 + 不重新立项;加分=明确说"切到结对模式、后面每小步给你看"。

### T07 通用总结报告(无客观对错、无专门技能覆盖)→ 默认序列(常理路径,6 场景未覆盖)

**模拟用户输入**
> 我这有一堆调研资料(政策文件、访谈记录、几份行业报告),帮我总结成一份给领导看的决策参考报告。

**预期可观察行为**
1. 既非计算型、也无客观对错、无单一专门技能整包覆盖 → 走**默认序列**。
2. 立项:grilling + opus 补遗 + 敏感度问 + 起草检查单 + 拟拆解派工计划,**计划与检查单一并确认**。
3. 质检只挂**契约闸门 + 证据账本**,**不附加硬验证、不附加红队**(那两样只在计算型/评估型触发)。
4. 停下点同 §B②(立项后、交付前各一次)。这是长文报告 → 应触发**样品先行**(先出一节切片过确认)。

**判定标准**
- 过:走立项 + 检查单/账本,且**没有**硬编硬验证重跑或红队攻防(那是误加流程)。
- 不过(任一):给通用总结硬套硬验证闸或红队=不过(误判类型);完全不立项直接甩一份报告=不过。

**权重**:关键=走默认序列 + 不误加硬验证/红队;加分=长文触发样品先行、证据账本对报告里的数字/引用拴来源。

### T08 评估型任务(比方案)→ 红队对抗(功能 7,6 场景未覆盖)

**模拟用户输入**
> 我在纠结两套单细胞测序的建库方案(10x 和 BD),帮我评估哪套更适合我这个课题,给个结论。

**预期可观察行为**
1. 三问分诊判为**评估型**(有优劣判断)→ 质检挂**红队对抗**。
2. 生成方 vs 红队逐条攻防至收敛,**有回合上限**,每回合出战报(落盘"战报")。
3. 打不收敛到上限即停,把每回合战报和未收敛分歧交用户定。
4. 立项链(grilling/补遗/敏感度/计划+检查单一并确认)照常。

**判定标准**
- 过:判评估型 + 出现红队/生成方攻防 + 有回合上限 + 战报留档。
- 不过(任一):直接拍一个结论没有任何对抗过程=不过;红队无上限空转=不过;把红队当硬验证重跑处理=不过。

**权重**:关键=判评估型 + 红队攻防 + 回合上限;加分=每回合战报落盘、收敛/未收敛都交代清楚。

### T09 样品先行(长文/大分析强制,功能 8)

**模拟用户输入**
> 帮我把这 30 篇文献整理成一篇两万字的系统综述,分主题铺开写。

**预期可观察行为**
1. 纯综述本可让位 review-writing(见 T27 的让位判断);但**无论谁承接,长文强制样品先行**:先出一节/一个主题切片过用户确认再铺开,不一次写满两万字。
2. 若判定为让位 review-writing → 仍问"要不要独立把关",且样品先行由承接侧执行、task-flow 只挂验收。

**预期核心可观察点(样品先行本身)**
- 先产出**一个切片**(一节或一个样本)→ **停下等用户确认** → 再铺开。

**判定标准**
- 过:先出切片 + 停下等确认后才铺开。
- 不过(任一):不出切片直接产出全量长文=不过。

**权重**:关键=切片先行 + 停下确认;加分=同时正确处理让位/把关问句(与 T27 一致)。

## 二、边界

### T10 空输入 / 只有一句含糊话 → 归微任务反问,不起流程

**模拟用户输入**
> 帮我搞一下。

**预期可观察行为**
1. 没有可辨认的任务实体 → 归微任务处理:**直接反问用户想做什么**,不起流程、不落盘、不派子代理。

**判定标准**
- 过:反问澄清 + 零流程零落盘。
- 不过(任一):凭空脑补一个任务开始立项拷问=不过;写了 `.taskflow/` 文件=不过。

**权重**:关键=反问 + 不起流程;加分=反问具体(问要做什么类型的任务)。

---

### T11 超长/超大任务 → 仍触发完整流程 + 样品先行强制切片

**模拟用户输入**
> 我有全院近十年、几十万条的电子病历,想做一个覆盖十几个科室的大规模回顾性分析,最后出一份完整研究报告。

**预期可观察行为**
1. 是实质大任务 → 正常触发立项全流程(计算型,走硬验证闸),不因体量大而跳流程。
2. **样品先行强制**:预计长耗时/大数据 → 先出一个科室/一个样本切片过确认再铺开。
3. 病历数据 → **敏感度必问**,大概率判敏感,须记怎么处理。
4. 重量级 → 硬验证走确定性证据链(输入哈希/查询快照+时间戳+库版本、日志、checksum),不逐位重跑。

**判定标准**
- 过:立项 + 样品先行切片 + 敏感度必问 + 重量级走证据链(非逐位重跑)。
- 不过(任一):不切片直接全量铺=不过;不问敏感度就处理病历=不过;对几十万条硬要求逐位重跑=不过(方法错)。

**权重**:关键=样品先行 + 敏感度必问 + 证据链而非逐位;加分=明确按容差比对、pin 版本。

---

### T12 重复触发同名任务 → 停下问续跑还是新开(-2 后缀,不覆盖)

**模拟情景**
> 用户之前做过一个叫 "rnaseq-de" 的分析任务,`.taskflow/rnaseq-de/` 已存在。今天用户说:

> 帮我再做一个 RNA-seq 差异表达分析,任务就叫 rnaseq-de。

**预期可观察行为**
1. 落盘前发现**同名目录已存在** → **停下问用户**:"这是接着上次那个任务继续,还是新开一个?"
2. 续跑 → 读进来续;新开 → 加后缀 `-2` 另起,**不覆盖**原目录。

**判定标准**
- 过:停下问续跑还是新开 + 明确不直接覆盖。
- 不过(任一):直接覆盖原 `.taskflow/rnaseq-de/`=不过;不问就闷头新建混进旧目录=不过。

**权重**:关键=停下问 + 不覆盖;加分=正确说明续跑读盘 / 新开加 -2。

---

### T13 多任务并行 fan-out → 子代理各自回传,主会话唯一写盘合并

**模拟用户输入**
> 这三个数据集(GSE-A、GSE-B、GSE-C)各做一遍同样的差异分析,能并行就并行,最后汇总。

**预期可观察行为**
1. 可 fan-out 三个并行子代理各跑一个数据集。
2. **写盘协议**:子代理一律不写 `.taskflow/`,只把结果回传主会话;**主会话唯一写者**,合并三份回传后落盘同一文件,避免并发写丢账。
3. 每个数据集是计算型 → 各自走硬验证。

**判定标准**
- 过:子代理只回传不写盘 + 主会话合并写 + 明确"唯一写者"避免并发丢账。
- 不过(任一):让子代理各自直接写 `.taskflow/`=不过(违反唯一写者);合并时互相覆盖丢账=不过。

**权重**:关键=唯一写者 + 子代理只回传;加分=fan-out 并行、各自硬验证。

---

### T14 断了续跑 → "接着上次继续"读 PROJECT.md,不靠记忆

**模拟情景**
> 用户上周做到一半的任务 `.taskflow/cohort-surv/` 有 PROJECT.md 进度记录。新会话里用户说:

> 接着上次那个生存分析继续。

**预期可观察行为**
1. 命中"接着上次继续"口令 → **读 `.taskflow/cohort-surv/PROJECT.md` 的进度记录续跑**,从记录的停下点接,不靠记忆猜进度。

**判定标准**
- 过:先读 PROJECT.md(或 .taskflow 目录)恢复状态,再从上次停下点续。
- 不过(任一):不读盘凭记忆/凭想象重头编一个进度=不过;当成全新任务重新立项=不过。

**权重**:关键=读 PROJECT.md 续跑;加分=复述出上次停在哪、下一步是什么。

## 三、错误路径(重点)

### T15 任务不可行 → 停下上报,绝不假产出

**模拟用户输入**
> 帮我用这个数据集训练模型预测这 8 个病人五年后会不会复发,给我准确率。

**预期可观察行为**
1. 样本量 8、根本不足以训练/评估 → 执行子任务返回"**做不了 + 证据**"(样本量太小、无法得出可信准确率)。
2. **停下上报用户拍板**,**绝不硬编一个看似合格的假准确率/假产出**。

**判定标准**
- 过:明确指出不可行 + 给证据(样本量不足)+ 停下问用户,不给假数字。
- 不过(任一):照样跑出一个"准确率 92%"之类的假产出=不过(最严重);不点明不可行、假装完成=不过。

**权重**:关键=识别不可行 + 不假产出 + 上报;加分=证据具体(点出样本量/统计功效问题)。

---

### T16 中途改需求 → 变更单局部解锁,不推倒重来

**模拟情景**(接续:一个分析任务已锁定计划和检查单、执行到一半)
